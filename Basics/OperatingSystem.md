OS的基本特征：并发&共享
线程&进程的区别{
线程：基本的CPU执行单元，独立调度和分派的基本单位，本身不拥有资源，共享进程的全部资源
进程：除CPU以外系统资源的分配单位
}
处理机调度的层次{
作业调度(高级)：内存<->辅存（调入一次调出一次）
内存调度(中级)：内存<->外存（挂起暂不能运行的）
进程调度(低级)：分配CPU
}
典型调度算法：FCFS、SJF、优先级调度、高响应比优先、时间片轮转
临界资源：一次仅允许一个进程使用的资源
同步{
定义：多个进程为共同完成某个任务而要协调工作次序的制约关系（源于相互合作，直接制约关系）
}
互斥{
定义：争用临界资源，一进一等（间接制约关系）
}
死锁{
定义：多个进程因竞争资源而造成的一种互相等待的僵局
条件：互斥（独享资源）、不剥夺（只能主动释放）、请求和保持、循环等待
例子：两车过桥（单车宽度，只能前进）
}
死锁预防：破坏四个必要条件之一
死锁避免：防止进入不安全状态<银行家算法，破坏了循环等待>
OS中的顺序存取&随机存取{
}
{
覆盖：同一个程序或进程中，无序全部装入即可运行[历史]
交换：不同进程之间（内存调度、虚拟内存）
}
内存管理{
连续分配
非连续分配（基本分页、分段、段页）
页是信息的物理单位，段是信息的逻辑单位
}
局部性原理{
时间：某条指令执行(数据访问)后，不久后可能再次执行(访问)【循环、堆栈】
空间：访问了某存储单元后，附近单元也将被访问【顺序存放，顺序执行】
}
虚拟内存{
定义：基于局部性原理，在程序装入时，可以将程序的一部分装入内存，而将其余部分留在外存，就可以启动程序执行。在程序执行过程中，当所访问的信息不在内存时，由操作系统将所需要的部分调入内存,然后继续执行程序。另一方面，操作系统将内存中暂时不使用的内容换出到外存上，从而腾出空间存放将要调入内存的信息。这样，系统好像为用户提供了一个比实际内存大得多的存储器，称为虚拟存储器。
如何加速：加内存，减少多道数，换更快的磁盘
}
文件：信息的集合，是I/O的基本单位
文件的逻辑结构{
无结构：流式
有结构：顺序、索引、索引顺序
}
文件共享{
基于索引结点（硬链接）
基于符号链（软链接[多次读盘]）
}
文件系统实现(物理){
目录实现：线性列表or哈希
文件实现：连续分配、链接分配、索引分配
}
操作系统中顺序存储跟随机储存的区别 {
在计算机中用一组地址连续的存储单元依次存储线性表的各个数据元素,称作线性表的顺序存储结构.　
顺序存储结构是存储结构类型中的一种，该结构是把逻辑上相邻的节点存储在物理位置上相邻的存储单元中，结点之间的逻辑关系由存储单元的邻接关系来体现。
在计算机中用一组任意的存储单元存储线性表的数据元素(这组存储单元可以是连续的,也可以是不连续的).
它不要求逻辑上相邻的元素在物理位置上也相邻.因此它没有顺序存储结构所具有的弱点,但也同时失去了顺序表可随机存取的优点.
}
I/O方式{
程序查询（仅软件）
程序中断（软硬结合）
DMA方式（仅硬件）
通道方式（软硬结合，专门负责I/O的处理机）
}
磁盘高速缓存(Disk Cache)≠Cache：逻辑上的磁盘，物理上驻留在内存的盘快
